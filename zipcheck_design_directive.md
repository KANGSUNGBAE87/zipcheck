# ZipCheck UI 전면 개편 지시문

기존 React + TypeScript + Vite 앱의 데이터 구조와 localStorage 동작은 유지하고, UI/UX만 “실무용 부동산 거래 대시보드”로 전면 개편한다.

## 핵심 목표
1. 첫 화면에서 `현재 거래`, `진행률`, `지금 해야 할 1개 항목`을 3초 안에 파악하게 한다.
2. 단순한 흰색 카드 나열이 아니라 신뢰감 있는 업무 도구의 시각 계층을 만든다.
3. 4단계 참고자료는 완료 항목과 분리해 진행률 계산에 포함하지 않는다.
4. 모바일에서는 목록과 상세를 같은 긴 페이지에 쌓지 말고 별도 화면처럼 전환한다.
5. 서버·로그인·동기화는 추가하지 않으며 기존 로컬 저장 원칙을 유지한다.

## 정보 구조
- `AppHeader`: ZIPCHECK 로고, 로컬 저장 상태, 언어 전환, 설정 메뉴
- `DashboardSummary`: 진행 중 거래 / 남은 확인 / 완료 거래
- `DealSidebar`: 검색, 상태 필터, 새 거래, 거래 카드
- `DealOverview`: 거래명, 거래 유형, 사용자 역할, 잔금일, 실행 항목 진행률
- `PhaseStepper`: 계약 전 → 계약 당일 → 잔금 준비 → 잔금일 참고 → 계약 후
- `NextActionCard`: 현재 가장 먼저 해야 할 항목 1개를 강하게 강조
- `PhaseChecklist`: 현재 단계는 펼침, 완료 단계와 미래 단계는 접힘
- `ReferenceLibrary`: 참고 전용 4개 항목을 별도 영역으로 분리
- `MemoDrawer`: 거래 전체/항목별 메모를 우측 패널 또는 모바일 바텀시트로 제공
- `ActivityTimeline`: 내부 이벤트 코드 대신 사람이 읽는 문장과 시각 표시

## 비주얼 토큰
- 배경 `#F4F6F7`
- 본문 `#17212B`
- 브랜드 네이비 `#173F4F`
- 실행/완료 그린 `#1B7059`
- 완료 배경 `#E7F4EE`
- 참고자료 샌드 `#F7F1E7`
- 주의 앰버 `#B76A12`, 배경 `#FFF5E7`
- 경계선 `#DDE4E9`
- 카드 반경 18~22px, 컨트롤 10~12px
- 그림자는 한 단계만 사용: `0 18px 48px rgba(23,33,43,.08)`
- Pretendard 우선, 시스템 폰트 폴백
- 본문 최소 14px, 보조문구 최소 12px, 터치 영역 최소 44px

## 레이아웃
- 데스크톱: 최대 1280px, 좌측 320px 거래 목록 + 우측 상세
- 821~1024px: 좌측 280px + 상세
- 820px 이하: 거래 목록/상세 분리, 하단 탭 `거래 · 체크리스트 · 메모`
- 단계 바는 모바일에서 가로 스크롤 허용
- 핵심 완료 CTA는 모바일 하단에 sticky 처리

## 반드시 수정할 로직
```ts
const actionableAlerts = alerts.filter((alert) => alert.status !== 'reference');
const doneCount = actionableAlerts.filter((alert) => alert.status === 'done').length;
const progressTotal = actionableAlerts.length; // 12
const progressPercent = Math.round((doneCount / progressTotal) * 100);
```
- 거래 카드에도 완료 상태 클래스가 실제로 적용되도록 한다.
- `history`의 `phase_viewed`, `alert_viewed` 같은 내부 문자열을 그대로 표시하지 않는다.
- 통계 `3 / 7`처럼 의미 없는 숫자 묶음을 쓰지 말고 각각 라벨이 있는 카드로 표현한다.

## 컴포넌트 상태
- hover, pressed, focus-visible, disabled, completed, current, reference 상태를 모두 설계한다.
- 키보드 포커스는 2px 외곽선으로 분명하게 보인다.
- 완료 직후 undo는 토스트로 5초간 노출한다.
- 참고자료는 체크박스 대신 문서/연락/확인 아이콘과 `참고 보기` 버튼을 쓴다.
- 빈 화면에는 일러스트, 설명, `새 거래 시작`, `샘플 거래로 둘러보기`를 제공한다.

## 문구 예시
- 메인 제목: `진행 중인 거래를 이어서 확인하세요`
- 설명: `중요한 확인 항목을 단계별로 정리하고, 놓친 업무 없이 마무리합니다.`
- 다음 행동: `지금 확인할 항목 · 1개 남음`
- 개인정보 안내: `거래명과 메모는 서버로 전송하지 않고 현재 기기에만 저장됩니다.`
- 진행률: `8 / 12 완료`
- 참고 단계: `완료에 포함되지 않는 참고자료 4개`

## 구현 제약
- 기존 `CaseItem`, `AlertItem`, `Memo`, `TEMPLATE` 타입을 유지한다.
- 외부 UI 프레임워크는 추가하지 않는다.
- 아이콘은 인라인 SVG 또는 경량 아이콘 패키지 한 개만 사용한다.
- 기존 테스트를 깨지 않으며 진행률/완료 계산 테스트를 추가한다.
- 390×844, 768×1024, 1440×900 뷰포트에서 확인한다.
- 색상만으로 상태를 구분하지 말고 아이콘·문구를 함께 사용한다.
